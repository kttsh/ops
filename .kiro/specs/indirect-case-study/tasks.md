# Implementation Plan

- [ ] 1. 型定義とAPIクライアントの基盤構築
- [ ] 1.1 人員計画ケース・月次人員数の型定義とZodスキーマを作成する
  - 人員計画ケース（ケース名、説明、BUコード、プライマリフラグ、論理削除）のAPI応答型を定義する
  - 月次人員数（BUコード、年月、人員数）のAPI応答型を定義する
  - 作成・更新・一括更新用のZodバリデーションスキーマを定義する（ケース名1〜100文字、説明0〜500文字、人員数0以上の整数）
  - 共通型（PaginatedResponse、SingleResponse）を再エクスポートする
  - _Requirements: 2.1, 2.2, 2.8, 3.1, 3.3, 3.4_

- [ ] 1.2 (P) キャパシティシナリオ・月次キャパシティの型定義とZodスキーマを作成する
  - キャパシティシナリオ（シナリオ名、労働時間、プライマリフラグ、論理削除）のAPI応答型を定義する
  - 月次キャパシティ（BUコード、年月、キャパシティ値）のAPI応答型を定義する
  - キャパシティ計算リクエスト・レスポンスの型を定義する
  - 作成・更新用のZodスキーマを定義する（シナリオ名1〜100文字、労働時間0超〜744以下、デフォルト160.00）
  - _Requirements: 4.1, 4.2, 4.7, 4.8, 5.1, 5.2_

- [ ] 1.3 (P) 間接作業ケース・比率・月次工数の型定義とZodスキーマを作成する
  - 間接作業ケース（ケース名、BUコード、プライマリフラグ、論理削除）のAPI応答型を定義する
  - 間接作業比率（作業種類コード、年度、比率0〜1）のAPI応答型を定義する
  - 月次間接作業工数（BUコード、年月、工数、source区分）のAPI応答型を定義する
  - 比率入力用Zodスキーマを定義する（UI上0〜100%で入力し送信時に0〜1変換）

- [ ] 1.4 (P) フロントエンド計算関連の型定義を作成する

  - 型のインデックスファイルで全エクスポートを集約する
  - _Requirements: 8.1, 9.2, 9.4, 13.1_

- [ ] 4. カスタムフックの実装
- [ ] 4.1 キャパシティ計算フックを実装する

  - 計算中のローディング状態とエラー状態を管理する
  - 計算結果（月次キャパシティ配列）を保持する

- [ ] 4.2 (P) 間接工数計算フックを実装する
  - キャパシティ結果と間接作業比率から月次間接工数を算出するフロントエンド計算ロジックを実装する
  - 年度判定（4月〜翌3月）に基づき、該当年度の全比率を合計してキャパシティに乗算する
  - 作業種類ごとの按分計算（キャパシティ × 個別比率）を実装する（表示用、DB非保存）
  - 計算結果に `source: 'calculated'` を設定する
  - _Requirements: 8.1, 8.2, 8.3, 9.4, 9.5_

- [ ] 4.3 (P) 未保存変更検知フックを実装する

  - window の beforeunload イベントでブラウザ閉じを検知する機能を実装する
  - isDirty フラグ（boolean）を外部から受け取り、遷移ガードの有効/無効を制御する
  - ダイアログ表示状態と3つのアクション（キャンセル・保存せず移動・保存）のハンドラを返す
  - _Requirements: 12.1, 12.2, 12.3, 12.4, 12.5_

- [ ] 4.4 (P) Excelエクスポートフックを実装する
  - xlsx パッケージを動的インポートで読み込み、計算結果テーブルデータからExcelファイルを生成する
  - ヘッダーに選択中の条件（人員計画ケース名・シナリオ名・間接作業ケース名）を出力する
  - 人員数・キャパシティ・間接内訳・間接合計・直接作業可能時間の全行を含める
  - writeFileXLSX でバンドルサイズを最適化する
  - _Requirements: 13.1, 13.2, 13.3_

- [ ] 4.5 ページ全体の状態管理フックを実装する

  - 設定用年度と結果表示用年度の切替を管理する
  - dirty統合を実装する：月次人員数dirty、間接作業比率dirty、間接工数結果dirtyをOR結合する
  - BU変更時に全ローカル状態をリセットする
  - 内部でキャパシティ計算フック・間接工数計算フックを利用する
