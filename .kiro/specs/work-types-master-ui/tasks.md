# Implementation Plan

- [x] 1. 作業種類 feature モジュール基盤
- [x] 1.1 型定義と Zod スキーマの定義
  - 作業種類の API レスポンス型（workTypeCode, name, displayOrder, color, createdAt, updatedAt, deletedAt）を定義する
  - 共通レスポンス型（PaginatedResponse, SingleResponse, ProblemDetails）を定義する
  - 作成用 Zod スキーマ（workTypeCode: 必須・最大20文字・英数字ハイフンアンダースコア、name: 必須・最大100文字、displayOrder: 0以上の整数・デフォルト0、color: #RRGGBB 形式・null 許容）を定義する
  - 更新用 Zod スキーマ（name: 必須・最大100文字、displayOrder: 0以上の整数・任意、color: #RRGGBB 形式・null 許容）を定義する
  - 一覧画面の search params 用 Zod スキーマ（検索文字列、削除済み含むフラグ）を定義する
  - _Requirements: 11.3, 4.2, 4.4, 5.3, 8.2, 8.3_

- [x] 1.2 API クライアントの実装
  - 薄い fetch ラッパーと ApiError クラスを実装する
  - 6 つの API 呼び出し関数を実装する: 一覧取得（GET /work-types）、単一取得（GET /work-types/:code）、作成（POST /work-types）、更新（PUT /work-types/:code）、削除（DELETE /work-types/:code）、復元（POST /work-types/:code/actions/restore）
  - レスポンスの JSON パースと型アサーション、RFC 9457 ProblemDetails エラーの throw を一箇所に集約する
  - 一覧取得では `filter[includeDisabled]` クエリパラメータを組み立てる
  - _Requirements: 1.1, 3.1, 4.5, 5.4, 6.2, 7.3_

- [x] 1.3 (P) TanStack Query の queryOptions と mutation フックの定義
  - Query Key Factory パターンで `workTypeKeys`（all, lists, list, details, detail）を定義する
  - `workTypesQueryOptions`（一覧取得）と `workTypeQueryOptions`（単一取得）を queryOptions パターンで定義する
  - 作成・更新・削除・復元の 4 つの useMutation フックを定義する
  - 各 mutation の `onSuccess` で作業種類一覧のキャッシュを無効化する
  - _Requirements: 1.1, 3.1, 4.5, 4.6, 4.7, 4.8, 5.4, 5.5, 5.6, 5.7, 6.2, 6.3, 6.4, 6.5, 7.3, 7.4, 7.5_

- [x] 1.4 (P) feature エクスポートインデックスの作成
  - 外部に公開する型、スキーマ、API フック、コンポーネントをバレルエクスポートする
  - _Requirements: 11.4_

- [x] 2. 作業種類一覧画面
- [x] 2.1 カラム定義の実装（カラースウォッチ列含む）
  - テーブルのカラム定義を作成する: 作業種類コード、名称、カラー、表示順、作成日時、更新日時
  - カラーカラムに色見本（小さな丸い要素に backgroundColor を設定）を表示し、color が null の場合は「-」テキストを表示する
  - 削除済みレコードにステータスバッジ（「有効」/「削除済み」）を表示するセルレンダラーを実装する
  - 削除済み行に「復元」ボタンを条件付きで表示する
  - 行クリック時に詳細画面へ遷移するリンク動作を設定する
  - テーブルヘッダーにソートインジケーターを表示する
  - _Requirements: 1.2, 1.3, 2.4, 7.1, 3.1, 10.5_

- [x] 2.2 DataTable コンポーネントの構築
  - TanStack Table の `useReactTable` を使用した DataTable コンポーネントを構築する
  - クライアントサイドのソート機能（`SortingState`）を実装する
  - クライアントサイドのグローバルフィルタ（テキスト部分一致検索）を実装する
  - shadcn/ui の Table コンポーネントを使用してレンダリングする
  - テーブル行のホバー時にスムーズなハイライトトランジションを適用する
  - データ取得中のローディング状態と、エラー発生時のエラーメッセージ表示を実装する
  - テーブルがビューポートに収まらない場合の水平スクロールを対応する
  - _Requirements: 1.4, 1.5, 1.6, 2.2, 9.5_

- [x] 2.3 DataTableToolbar コンポーネントの構築
  - テーブル上部のツールバーを構築する
  - IME 対応のデバウンス付き検索入力欄（作業種類コードまたは名称でフィルタリング）を配置する
  - 「削除済みを含む」トグルスイッチを配置する
  - 「新規登録」ボタンを配置し、新規登録画面へのリンクを設定する
  - _Requirements: 2.1, 2.3, 4.1_

- [x] 2.4 一覧画面ルートの実装
  - `/master/work-types` のルートファイルを作成する
  - search params（検索文字列、削除済み含むフラグ）を Zod でバリデーションする
  - `workTypesQueryOptions` を使用して API からデータを取得する
  - 「削除済みを含む」トグルの状態変更時に API を再取得する
  - 検索条件と削除済みトグルの状態を URL search params に反映する
  - 削除済みレコードの行に透明度を下げた視覚的区別を適用する
  - DataTable、DataTableToolbar、カラム定義を組み合わせてページコンポーネントを構成する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 2.1, 2.2, 2.3, 2.4, 8.1, 8.2, 8.3_

- [x] 3. 作業種類詳細画面
- [x] 3.1 詳細画面ルートの実装
  - `/master/work-types/$workTypeCode` のルートファイルを作成する
  - `workTypeQueryOptions` を使用して単一の作業種類を取得・表示する
  - 作業種類コード、名称、カラー（カラースウォッチ付き）、表示順、作成日時、更新日時をカード形式で表示する
  - 「編集」ボタン（編集画面へのリンク）と「削除」ボタン（確認ダイアログ起動）を配置する
  - パンくずリスト（「作業種類一覧 > [名称]」）で戻るナビゲーションを提供する
  - 作業種類が存在しない場合の 404 エラー画面を設定する
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5_

- [x] 4. 作業種類新規登録・編集画面
- [x] 4.1 WorkTypeForm 共通フォームコンポーネントの構築（カラーピッカー含む）
  - TanStack Form を使用した新規登録・編集共通のフォームコンポーネントを構築する
  - `mode` プロパティ（`'create'` / `'edit'`）で表示を切り替える
  - 作業種類コード（create モードのみ編集可能）、名称、表示順の入力フィールドを提供する
  - カラーフィールドはネイティブカラーピッカーとテキスト入力（#RRGGBB）を横並びで配置する
  - カラーが null の場合はデフォルト値を表示し、「クリア」ボタンで null に戻せるようにする
  - Zod スキーマによるフィールドレベルのリアルタイムバリデーションを実装する
  - バリデーションエラーを各入力フィールドの直下にインラインで表示する
  - 送信中はボタンを無効化し、スピナーアイコンを表示する
  - フォーム画面の入力フィールド幅を適切に制限する
  - _Requirements: 4.2, 4.3, 4.4, 5.2, 5.3, 10.3, 10.4_

- [x] 4.2 (P) 新規登録画面ルートの実装
  - `/master/work-types/new` のルートファイルを作成する
  - WorkTypeForm を `mode: 'create'` で表示する
  - フォーム送信時に `useCreateWorkType` mutation を呼び出す
  - 作成成功時に一覧画面へリダイレクトし、「保存しました」Toast を表示する
  - 409 Conflict 時に「同一コードの作業種類が既に存在します」エラー Toast を表示する
  - 422 バリデーションエラー時にフォームにエラー内容を表示する
  - パンくずリスト（「作業種類一覧 > 新規登録」）を配置する
  - _Requirements: 4.1, 4.5, 4.6, 4.7, 4.8, 10.1, 10.2_

- [x] 4.3 (P) 編集画面ルートの実装
  - `/master/work-types/$workTypeCode/edit` のルートファイルを作成する
  - `workTypeQueryOptions` で既存データを取得し、フォームの初期値に設定する（color 値含む）
  - WorkTypeForm を `mode: 'edit'` で表示する
  - フォーム送信時に `useUpdateWorkType` mutation を呼び出す
  - 更新成功時に詳細画面へリダイレクトし、「保存しました」Toast を表示する
  - 404 Not Found 時に「作業種類が見つかりません」エラー Toast を表示する
  - 422 バリデーションエラー時にフォームにエラー内容を表示する
  - _Requirements: 5.1, 5.2, 5.4, 5.5, 5.6, 5.7, 10.1, 10.2_

- [x] 5. 削除・復元機能
- [x] 5.1 削除確認ダイアログと削除処理の実装
  - shadcn/ui AlertDialog を使用した削除確認ダイアログを構築する
  - 確認ダイアログで「削除」を選択した場合に `useDeleteWorkType` mutation を呼び出す
  - 削除成功時に一覧画面へリダイレクトし、「削除しました」Toast を表示する
  - 409 Conflict 時に「この作業種類は他のデータから参照されているため削除できません」エラー Toast を表示する
  - 404 Not Found 時に「作業種類が見つかりません」エラー Toast を表示する
  - 詳細画面の「削除」ボタンとダイアログを接続する
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5, 10.1, 10.2_

- [x] 5.2 (P) 復元確認ダイアログと復元処理の実装
  - shadcn/ui AlertDialog を使用した復元確認ダイアログを構築する
  - 確認ダイアログで「復元」を選択した場合に `useRestoreWorkType` mutation を呼び出す
  - 復元成功時にテーブルを再取得し、「復元しました」Toast を表示する
  - 409 Conflict 時にエラー Toast を表示する
  - 一覧画面の削除済み行の「復元」ボタンとダイアログを接続する
  - _Requirements: 7.1, 7.2, 7.3, 7.4, 7.5, 10.1, 10.2_
