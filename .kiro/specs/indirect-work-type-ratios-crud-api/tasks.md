# Implementation Plan

- [x] 1. Zod スキーマと型定義の作成
- [x] 1.1 間接作業種別配分比率のリクエストスキーマと型を定義する
  - 作成用スキーマ: workTypeCode（最大20文字の文字列、必須）、fiscalYear（整数、必須）、ratio（0以上1以下の数値、必須）を受け付ける
  - 更新用スキーマ: 全フィールドを任意（optional）にした部分更新スキーマを定義する
  - バルク Upsert 用スキーマ: items 配列（1件以上）で各要素は作成用と同じフィールドを含む
  - _Requirements: 7.1, 7.2, 7.3_

- [x] 1.2 (P) DB 行型と API レスポンス型を定義する
  - DB 行型（snake_case）: テーブルカラムに対応する7フィールド（id, case_id, work_type_code, fiscal_year, ratio, created_at, updated_at）
  - API レスポンス型（camelCase）: 同等フィールドで日時は文字列型、ratio は数値型
  - _Requirements: 8.3, 8.4, 8.5_

- [x] 2. Transform 関数の作成
- [x] 2.1 DB 行から API レスポンスへの変換関数を実装する
  - snake_case フィールドを camelCase に変換する
  - Date 型を ISO 8601 文字列に変換する
  - ratio を数値型のまま維持する
  - _Requirements: 8.3, 8.4, 8.5_

- [x] 3. Data 層の実装
- [x] 3.1 基本的な CRUD クエリ関数を実装する
  - 指定された間接作業ケースに紐づく全配分比率レコードを年度・作業種別順で取得する関数
  - ID 指定で単一レコードを取得する関数
  - 新規レコードを INSERT し、採番された ID で再取得して返却する関数
  - 指定フィールドのみ動的に UPDATE する関数（updated_at を自動更新）
  - ID 指定で物理削除する関数
  - _Requirements: 1.1, 2.1, 3.1, 4.1, 5.1_

- [x] 3.2 存在確認・重複チェック関数を実装する
  - 間接作業ケースの存在確認（deleted_at IS NULL 条件付き）
  - 作業種別マスタの存在確認（deleted_at IS NULL 条件付き）
  - 3カラム複合キー（ケースID + 作業種別コード + 年度）の重複チェック（更新時の除外 ID 対応付き）
  - _Requirements: 1.3, 2.4, 3.4, 3.5, 3.6, 4.4, 4.5, 5.3, 5.4_

- [x] 3.3 バルク Upsert 関数を実装する
  - SQL Server の MERGE 文を使用し、3カラム複合キーで一致判定する
  - 一致時は ratio と updated_at を更新、不一致時は新規挿入する
  - トランザクション内で全件を処理し、失敗時はロールバックする
  - 処理完了後に該当ケースの全レコードを返却する
  - _Requirements: 6.1, 6.2, 6.3, 6.6_

- [x] 4. Service 層の実装
- [x] 4.1 一覧取得・単一取得のビジネスロジックを実装する
  - 一覧取得: 親ケースの存在を確認し、存在しない場合は 404 を返す。存在する場合は全レコードを取得して変換する
  - 単一取得: レコードの存在と親ケースとの紐付きを確認し、不整合時は 404 を返す
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.2, 2.3, 2.4_

- [x] 4.2 新規作成のビジネスロジックを実装する
  - 親ケースの存在確認（404）
  - 作業種別マスタの存在確認（422）
  - 同一ケース + 作業種別 + 年度の重複チェック（409）
  - すべてのチェックを通過後にレコードを作成し、変換して返却する
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5, 3.6_

- [x] 4.3 更新のビジネスロジックを実装する
  - 対象レコードの存在と親ケースとの紐付き確認（404）
  - 更新後のフィールド値で複合キー重複チェック（変更があるフィールドのみ。409）
  - チェック通過後にレコードを更新し、変換して返却する
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5_

- [x] 4.4 (P) 削除のビジネスロジックを実装する
  - 対象レコードの存在と親ケースとの紐付き確認（404）
  - 物理削除を実行する
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

- [x] 4.5 バルク Upsert のビジネスロジックを実装する
  - 親ケースの存在確認（404）
  - リクエスト内の重複検出: 同一 workTypeCode + fiscalYear の組み合わせが複数ある場合は 422 を返す
  - Data 層のバルク Upsert を呼び出し、結果を変換して返却する
  - _Requirements: 6.1, 6.2, 6.3, 6.4, 6.5, 6.6_

- [x] 5. Route 層の実装とシステム統合
- [x] 5.1 HTTP エンドポイントを定義する
  - GET /（一覧取得）: 親ケース ID をパスパラメータから取得し、Service の一覧取得を呼び出して 200 で返却
  - GET /:indirectWorkTypeRatioId（単一取得）: 両パスパラメータを検証し、Service の単一取得を呼び出して 200 で返却
  - POST /（新規作成）: Zod バリデーションミドルウェアを適用し、Service の作成を呼び出して 201 + Location ヘッダで返却
  - PUT /:indirectWorkTypeRatioId（更新）: Zod バリデーションミドルウェアを適用し、Service の更新を呼び出して 200 で返却
  - DELETE /:indirectWorkTypeRatioId（削除）: Service の削除を呼び出して 204 で返却
  - PUT /bulk（バルク Upsert）: /bulk を個別 ID ルートより前に定義し、Zod バリデーションミドルウェアを適用して 200 で返却
  - パスパラメータ（indirectWorkCaseId, indirectWorkTypeRatioId）を正の整数として検証するヘルパーを使用する
  - _Requirements: 1.1, 1.2, 2.1, 2.2, 3.1, 3.3, 4.1, 4.3, 5.1, 6.1, 6.4, 7.4, 7.5, 8.1, 8.2_

- [x] 5.2 アプリケーションのルート登録に新規エンドポイントを追加する
  - `/indirect-work-cases/:indirectWorkCaseId/indirect-work-type-ratios` パスでルートをマウントする
  - 既存のルート登録パターンに従い、import と route 呼び出しを追加する
  - _Requirements: 1.1, 2.1, 3.1, 4.1, 5.1, 6.1_

- [x] 6. テストの実装
- [x] 6.1 型定義とスキーマのテストを作成する
  - 各 Zod スキーマで正常値、境界値、不正値のバリデーション結果を検証する
  - ratio の範囲（0.0000, 1.0000, 範囲外）、workTypeCode の長さ制限、fiscalYear の整数検証を含める
  - _Requirements: 7.1, 7.2, 7.3_

- [x] 6.2 (P) Transform 関数のテストを作成する
  - snake_case から camelCase への変換が正しいことを検証する
  - Date 型が ISO 8601 文字列に変換されることを検証する
  - ratio が数値型のまま維持されることを検証する
  - _Requirements: 8.3, 8.4, 8.5_

- [x] 6.3 Service 層のテストを作成する
  - Data 層をモックし、各メソッドの正常系・異常系を検証する
  - 親ケース不在時の 404、重複時の 409、不正 workTypeCode 時の 422 を含める
  - バルク操作内の重複検出ロジックを検証する
  - _Requirements: 1.3, 2.3, 2.4, 3.4, 3.5, 3.6, 4.4, 4.5, 5.3, 5.4, 6.4, 6.5, 6.6_

- [x] 6.4 Route 層の結合テストを作成する
  - Hono の app.request() で各エンドポイントのステータスコードとレスポンス構造を検証する
  - POST 時の 201 ステータスと Location ヘッダ、DELETE 時の 204 と空ボディを確認する
  - パスパラメータの不正値に対する 422 レスポンスを検証する
  - _Requirements: 1.1, 1.2, 2.1, 2.2, 3.1, 3.3, 4.1, 4.3, 5.1, 5.2, 6.1, 7.4, 7.5, 8.1, 8.2_
