# Implementation Plan

- [x] 1. BusinessUnitSingleSelector コンポーネントの作成
- [x] 1.1 チップ型シングルセレクトBU選択コンポーネントを実装する
  - BU一覧データを取得し、全BUをチップ（ピル型ボタン）として横並びに表示する
  - チップクリック時に該当BUを選択状態にし、他のBUの選択を解除するシングルセレクト動作を実装する
  - 選択中チップにアクティブスタイル、未選択チップにホバースタイルを適用する
  - ラベル「ビジネスユニット」と Building2 アイコンをチップ一覧の先頭に表示する
  - 「全選択」ボタンは含めない（シングルセレクトのため不要）
  - 山積ダッシュボードの BU 選択UIと同じ視覚パターン（チップ形状、色、アイコン・ラベル配置）に合わせる
  - 選択中BUコードと変更コールバックを外部から受け取る controlled component として設計する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 5.1, 5.2_
  - _Contracts: BusinessUnitSingleSelectorProps_

- [x] 2. シミュレーションページのレイアウト再構成とBU選択統合
- [x] 2.1 ヘッダーからドロップダウン型BU選択を削除し、ヘッダー直下にBU選択行を配置する
  - ヘッダー行からBUドロップダウン（Select コンポーネント群）とそのインポートを削除する
  - ヘッダー直下に山積ダッシュボードと同じレイアウトスタイルの独立行を追加する
  - BU選択行に BusinessUnitSingleSelector を配置し、既存の選択状態・変更ハンドラに接続する
  - BUチップクリック時にURL search params の bu パラメータを更新する動作を維持する
  - URL params に有効な bu パラメータがある場合は該当BUを選択状態で復元する
  - URL params に bu パラメータがない、または無効な場合は先頭BUを自動選択する
  - BU切替時のケース選択リセット・計算結果クリア・ケース一覧再取得は既存の useIndirectSimulation フックで実現済みであることを確認する
  - _Requirements: 1.1, 1.2, 1.3, 3.1, 3.2, 3.3, 4.1, 4.2, 4.3_
