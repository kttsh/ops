# Implementation Plan

- [x] 1. (P) ケースID解決ユーティリティの実装
  - URL検索パラメータから渡されたケースIDが有効なケースリスト内に存在するかを検証し、存在すればそのまま返す
  - 指定IDが無効（削除済み・存在しない）の場合はプライマリケースにフォールバックする
  - プライマリケースも存在しない場合は 0（未選択）を返す
  - ジェネリクスで3種のケース型（人員計画・稼働時間・間接作業）に汎用的に対応する
  - 既存のプライマリ検出ロジックを内部で再利用する
  - _Requirements: 1.3, 2.3, 3.3, 4.3_

- [x] 2. (P) ケースセレクタUIコンポーネントの作成
  - ケース一覧をドロップダウン形式で表示し、ユーザーが任意のケースを選択できるコンポーネントを作成する
  - プライマリケースには「★」バッジを表示し、どれが標準ケースかを視覚的に示す
  - ケースが0件の場合は「未登録」プレースホルダーを表示し、セレクタを無効化する
  - ローディング中・再計算中は操作を無効化する
  - ジェネリクスで3種のケース型に対応し、ID取得・ラベル取得・プライマリ判定の関数をpropsで受け取る
  - _Requirements: 1.1, 1.4, 2.1, 2.4, 3.1, 3.4_

- [x] 3. シミュレーションフックの改修
- [x] 3.1 フック入力のパラメータ化とケースID解決の組み込み
  - フックの入力にオプショナルなケースIDパラメータ（人員計画・稼働時間・間接作業）を追加する
  - ケースリスト取得後、Task 1 で作成したID解決ロジックを用いて有効なケースIDを決定する
  - 解決済みIDに基づいて月次データの取得クエリを切り替える
  - ケースリストデータを戻り値に追加し、セレクタUIから利用可能にする
  - 既存のプライマリID・プライマリ名の戻り値を解決済みIDに置換する
  - _Requirements: 1.3, 2.3, 3.3, 4.2, 4.3_

- [x] 3.2 再計算フローの更新
  - 再計算実行時に使用するケースIDを、プライマリ固定から解決済みIDに切り替える
  - キャパシティ計算APIへの入力（人員計画ケースID + 稼働時間シナリオID）を解決済みIDで呼び出す
  - 間接工数の算出・一括保存も解決済みの間接作業ケースIDを使用する
  - 3つの解決済みIDがすべて有効（> 0）の場合のみ再計算可能フラグを true にする
  - エラー時のトースト通知は既存の挙動を維持する
  - _Requirements: 5.1, 5.4, 1.5, 2.5, 3.5_

- [x] 4. 計算条件パネルとルートの統合
- [x] 4.1 計算条件パネルのセレクタ統合
  - 既存の読み取り専用ケース名表示を、Task 2 で作成したケースセレクタドロップダウンに置き換える
  - 3カラムグリッドレイアウトを維持し、各カラムに条件ラベル + セレクタを配置する
  - 人員計画セレクタにはBU紐づきのケース一覧、稼働時間には全シナリオ、間接作業にはBU紐づきのケース一覧を渡す
  - 再計算中はすべてのセレクタを無効化する
  - 3つすべてのケースが選択されている場合のみ再計算ボタンを有効にする
  - マスタ管理画面へのリンク（ExternalLink）は維持する
  - _Requirements: 1.2, 1.5, 2.2, 2.5, 3.2, 3.5, 5.2_

- [x] 4.2 URL検索パラメータとルートコンポーネントの接続
  - ルートの検索パラメータスキーマに3つのケースID（数値型、デフォルト0）を追加する
  - 検索パラメータからケースIDを取得し、シミュレーションフックに渡す
  - ケースセレクタの変更時にURL検索パラメータを更新する（ブラウザ履歴に反映）
  - BU変更時にBU依存のケースID（人員計画・間接作業）をリセットする（稼働時間はグローバルのため維持）
  - URLパラメータにケースIDが含まれる状態でのページ表示で選択状態を復元する
  - _Requirements: 4.1, 4.2, 4.3, 5.3_

- [x]* 5. テストカバレッジ
- [x]* 5.1 (P) ケースID解決ユーティリティのユニットテスト
  - URL paramで指定されたIDがリスト内に存在する場合にそのまま返されることを検証する
  - 指定IDがリスト内に存在しない場合にプライマリケースにフォールバックすることを検証する
  - プライマリケースも存在しない場合に0が返されることを検証する
  - ケースリストが空の場合の挙動を検証する
  - _Requirements: 1.3, 2.3, 3.3, 4.3_

- [x]* 5.2 (P) URL検索パラメータスキーマのバリデーションテスト
  - 正常な数値が正しくパースされることを検証する
  - 不正値（文字列、負数、小数）がデフォルト値（0）にフォールバックすることを検証する
  - パラメータ未指定時にデフォルト値が適用されることを検証する
  - _Requirements: 4.2, 4.3_
