# Implementation Plan

- [x] 1. 型定義とZodバリデーションスキーマの作成
- [x] 1.1 (P) チャートビューのZodスキーマとTypeScript型を定義する
  - 作成用スキーマ: ビュー名（必須、1〜100文字）、チャートタイプ（必須、1〜50文字）、開始年月（必須、YYYYMM形式6桁数字）、終了年月（必須、YYYYMM形式6桁数字）、デフォルトフラグ（任意、デフォルトfalse）、説明（任意、最大500文字、null許可）
  - 作成用スキーマに `.refine()` で開始年月 ≤ 終了年月のクロスフィールドバリデーションを追加（YYYYMM文字列の辞書順比較）
  - 更新用スキーマ: すべてのフィールドを任意とし、個別フィールドのバリデーションルールは作成用と同一。クロスフィールドバリデーションはサービス層で実施するためスキーマには含めない
  - 一覧取得クエリスキーマ: 既存のページネーションスキーマを拡張し、論理削除済みを含めるフィルタを追加
  - DB行型（snake_case）: chart_view_id, view_name, chart_type, start_year_month, end_year_month, is_default, description, created_at, updated_at, deleted_at
  - APIレスポンス型（camelCase）: chartViewId, viewName, chartType, startYearMonth, endYearMonth, isDefault, description, createdAt, updatedAt
  - Zodスキーマから作成・更新・クエリのTypeScript型を導出する
  - _Requirements: 7.4, 7.5, 8.1, 8.2, 8.3, 8.4, 8.5_

- [x] 1.2 (P) DB行からAPIレスポンスへの変換関数を実装する
  - snake_caseのDB行をcamelCaseのAPIレスポンスに変換する関数を作成
  - 日時フィールド（created_at, updated_at）をISO 8601文字列に変換
  - 外部キーのJOINフィールドがないため、直接的なフィールドマッピングのみ
  - _Requirements: 7.4, 7.5_

- [x] 2. データアクセス層の実装
- [x] 2.1 一覧取得と単一取得のデータアクセスを実装する
  - ページネーション付き一覧取得: chart_viewsテーブルへの単純なSELECT、ソフトデリートフィルタを適用、OFFSET/FETCHでページング
  - 単一取得（アクティブのみ）: IDを指定して返却、deleted_at IS NULLのレコードのみ対象
  - 単一取得（削除済み含む）: 復元処理や存在確認で使用するため、deleted_at条件なしで取得
  - 件数取得: ソフトデリートフィルタを考慮したCOUNTクエリ
  - すべてのクエリはパラメータ化し、SQLインジェクションを防止
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.2, 2.3_

- [x] 2.2 作成・更新のデータアクセスを実装する
  - 作成: ビュー名、チャートタイプ、開始年月、終了年月、デフォルトフラグ、説明をINSERTし、OUTPUT句で作成されたレコードを直接返却（JOINなしのためOUTPUTで十分）
  - 更新: 指定されたフィールドのみ動的にSET句を構成し、updated_atを現在日時に更新。UPDATE後にfindByIdで最新の行を返却
  - _Requirements: 3.1, 3.3, 4.1, 4.2_

- [x] 2.3 論理削除・復元のデータアクセスを実装する
  - 論理削除: deleted_atに現在日時を設定し、updated_atも更新。OUTPUT句で結果を返却
  - 復元: deleted_atをNULLに設定し、updated_atを現在日時に更新。OUTPUT句で結果を返却
  - 子テーブル（chart_view_project_items, chart_view_indirect_work_items）はCASCADE削除のため、参照チェック（hasReferences）は不要
  - _Requirements: 5.1, 5.2, 6.1, 6.2_

- [x] 3. サービス層の実装
- [x] 3.1 一覧取得・単一取得のビジネスロジックを実装する
  - 一覧取得: データ層から取得した結果を変換関数でAPIレスポンス形式にマッピングし、件数とともに返却
  - 単一取得: 指定IDのレコードが見つからない場合は404例外をスロー。見つかった場合は変換後に返却
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.2, 2.3_

- [x] 3.2 作成・更新のビジネスロジックを実装する
  - 作成: データ層のcreateを呼び出し、変換後に返却。年月の大小関係はZodスキーマの`.refine()`で検証済み
  - 更新: 既存レコードをfindByIdで取得。存在しなければ404例外。リクエストの開始年月・終了年月と既存値をマージし、マージ後の開始年月が終了年月より後であれば422例外をスロー。データ層のupdateを呼び出し、変換後に返却
  - _Requirements: 3.1, 3.4, 3.5, 4.1, 4.3, 4.4, 4.5_

- [x] 3.3 論理削除・復元のビジネスロジックを実装する
  - 削除: findByIdで存在確認し、存在しなければ404例外。論理削除を実行
  - 復元: 削除済み含むfindByIdで対象を確認。存在しなければ404例外。削除されていなければ404例外。削除済みの場合はrestoreを実行し変換後に返却
  - _Requirements: 5.1, 5.2, 5.3, 6.1, 6.2, 6.3_

- [x] 4. ルート層の実装とシステム統合
- [x] 4.1 CRUDエンドポイントを定義する
  - GET / : 一覧取得クエリスキーマでバリデーション後、サービス層を呼び出し、ページネーション付きレスポンスを返却
  - GET /:id : パスパラメータからIDを取得し、サービス層の単一取得を呼び出す
  - POST / : 作成スキーマでバリデーション後、サービス層を呼び出し、Locationヘッダ付きで201を返却
  - PUT /:id : 更新スキーマでバリデーション後、サービス層を呼び出し、200で返却
  - DELETE /:id : サービス層の削除を呼び出し、204 No Contentを返却
  - POST /:id/actions/restore : サービス層の復元を呼び出し、200で返却
  - Honoのメソッドチェーンでルートを定義し、RPC用にルート型をエクスポート
  - _Requirements: 1.1, 2.1, 3.1, 3.2, 4.1, 5.1, 6.1, 7.1, 7.2, 7.3, 8.1, 8.2, 8.3, 8.5_

- [x] 4.2 アプリケーションにルートをマウントする
  - エントリーポイントにchart-viewsルートをインポートし、`/chart-views`パスでマウント
  - 既存のグローバルエラーハンドラとRFC 9457レスポンス形式がそのまま適用されることを確認
  - _Requirements: 7.3_

- [x] 5. テストの実装
- [x] 5.1 データ層のユニットテストを作成する
  - mssqlのgetPool/request/input/queryをモックし、各データアクセスメソッドのSQL実行を検証
  - findAll: SQLの実行、ページネーションオフセット計算、ソフトデリートフィルタの切り替え
  - findById / findByIdIncludingDeleted: パラメータ化クエリの正しいID渡し
  - create: INSERT + OUTPUT句で返却される行の検証
  - update: 動的SET句の生成（指定フィールドのみ更新される）
  - softDelete / restore: deleted_atの操作
  - _Requirements: 9.1, 9.4_

- [x] 5.2 サービス層のユニットテストを作成する
  - データ層と変換関数をモックし、ビジネスロジックを検証
  - findAll: データ層呼び出しと変換適用
  - findById: 正常系の返却と、存在しない場合の404例外
  - create: 正常系の検証
  - update: 部分更新、年月クロスフィールドバリデーション（既存値マージで開始 > 終了の場合の422例外）、404例外
  - delete: 正常系の削除と404例外
  - restore: 未削除の場合の404例外、削除済みの場合の正常復元
  - _Requirements: 9.1, 9.3_

- [x] 5.3 ルート層のユニットテストを作成する
  - サービス層をモックし、app.request()でHTTPリクエストをシミュレート
  - GET / : 200レスポンス、ページネーションメタデータ、空リスト
  - GET /:id : 200レスポンス、404エラー
  - POST / : 201レスポンスとLocationヘッダ、422バリデーションエラー（必須項目欠落、型不正、範囲外値、年月大小関係不正）
  - PUT /:id : 200レスポンス、404エラー、422バリデーションエラー
  - DELETE /:id : 204レスポンス、404エラー
  - POST /:id/actions/restore : 200レスポンス、404エラー
  - _Requirements: 9.1, 9.2, 9.3, 9.4_
