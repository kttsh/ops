# Implementation Plan

- [x] 1. Zodスキーマと型定義を作成する
- [x] 1.1 チャートビュー間接作業項目の型とスキーマを定義する
  - DB Row型（snake_case）とAPI Response型（camelCase）を定義する
  - JOINで取得する間接作業ケース名フィールドをRow型に含める
  - 作成用スキーマを定義する：間接作業ケースID（必須・正の整数）、表示順序（任意・0以上・デフォルト0）、表示フラグ（任意・デフォルトtrue）
  - 更新用スキーマを定義する：表示順序（任意・0以上）、表示フラグ（任意）
  - _Requirements: 3.3, 4.2, 6.4, 7.2, 7.3_

- [x] 2. Transform関数を作成する
- [x] 2.1 (P) DB Row型からAPIレスポンス型への変換関数を実装する
  - snake_caseからcamelCaseへのフィールド名変換を行う
  - 日時フィールドをISO 8601形式の文字列に変換する
  - JOINで取得した間接作業ケース名をcamelCaseフィールドにマッピングする
  - BIT型フィールド（is_visible）のboolean変換を適切に処理する
  - _Requirements: 6.4, 6.5_

- [x] 3. Data層のDBクエリを実装する
- [x] 3.1 一覧取得と単一取得のクエリを実装する
  - 指定チャートビューに紐づく全項目を取得するクエリを実装する
  - 間接作業ケーステーブルとJOINして間接作業ケース名を含める
  - 表示順序の昇順でソートする
  - IDによる単一レコード取得クエリを実装する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2_

- [x] 3.2 作成・更新・削除のクエリを実装する
  - 新規レコードを挿入し、生成されたIDで再取得するクエリを実装する
  - 指定フィールドのみを更新するクエリを実装する（updated_atも更新）
  - IDによる物理削除クエリを実装する
  - _Requirements: 3.1, 4.1, 5.1_

- [x] 3.3 バリデーション用のヘルパークエリを実装する
  - チャートビューの存在確認クエリを実装する（論理削除済みは除外）
  - 間接作業ケースの存在確認クエリを実装する（論理削除済みは除外）
  - 同一チャートビュー内での間接作業ケースIDの重複チェッククエリを実装する
  - _Requirements: 1.4, 1.5, 3.4, 3.5, 3.6_

- [x] 4. Service層のビジネスロジックを実装する
- [x] 4.1 一覧取得と単一取得のロジックを実装する
  - 親チャートビューの存在を確認し、不存在時は404エラーを返す
  - 一覧取得では全項目を変換して返却する
  - 単一取得では項目の存在確認とチャートビューIDの一致確認を行う
  - _Requirements: 1.1, 1.4, 1.5, 2.1, 2.3, 2.4_

- [x] 4.2 作成ロジックを実装する
  - 親チャートビューの存在確認を行う
  - 間接作業ケースの存在確認を行い、不存在時は422エラーを返す
  - 同一チャートビュー内での重複を検証し、重複時は409エラーを返す
  - レコード作成後、JOINデータを含むレスポンスを返却する
  - _Requirements: 3.1, 3.4, 3.5, 3.6_

- [x] 4.3 更新と削除のロジックを実装する
  - 更新時は項目の存在確認とチャートビューIDの一致確認を行う
  - 削除時は項目の存在確認とチャートビューIDの一致確認を行い、物理削除する
  - すべてのエラーはRFC 9457 Problem Details形式のHTTPExceptionで返す
  - _Requirements: 4.1, 4.3, 4.4, 5.1, 5.2, 5.3_

- [x] 5. ルーターとエンドポイントを実装し、システムに統合する
- [x] 5.1 CRUDエンドポイントを定義する
  - 一覧取得（GET /）、単一取得（GET /:id）、作成（POST /）、更新（PUT /:id）、削除（DELETE /:id）を定義する
  - パスパラメータのバリデーション関数を定義する
  - 作成・更新リクエストボディにZodバリデーションを適用する
  - 作成成功時にLocationヘッダを設定する
  - エラーレスポンスはRFC 9457形式で返却する
  - _Requirements: 1.1, 2.1, 3.1, 3.2, 3.7, 4.1, 4.5, 5.1, 6.1, 6.2, 6.3, 7.1, 7.4_

- [x] 5.2 メインアプリケーションにルートを登録する
  - ネストURLパターンでルーターをメインアプリに登録する
  - 既存のチャートビュールートとの整合性を確認する
  - _Requirements: 1.1_

- [x] 6. ユニットテストを作成する
- [x] 6.1 正常系のCRUD操作テストを作成する
  - 一覧取得（表示順序ソート確認、間接作業ケース名含有確認）のテストを作成する
  - 単一取得（間接作業ケース名含有確認）のテストを作成する
  - 新規作成（201レスポンス、Locationヘッダ確認）のテストを作成する
  - 更新（200レスポンス、フィールド反映確認）のテストを作成する
  - 物理削除（204レスポンス確認）のテストを作成する
  - _Requirements: 8.1_

- [x] 6.2 バリデーションエラーのテストを作成する
  - 必須項目欠落時の422エラーを検証する
  - 型不正（文字列を数値フィールドに渡す等）時の422エラーを検証する
  - 範囲外値（負のdisplayOrder等）時の422エラーを検証する
  - エラーレスポンスのerrors配列にフィールドごとの詳細が含まれることを検証する
  - _Requirements: 8.2_

- [x] 6.3 エラーケースのテストを作成する
  - 存在しないチャートビューIDでの404エラーを検証する
  - 存在しない項目IDでの404エラーを検証する
  - チャートビューIDの不一致での404エラーを検証する
  - 同一チャートビュー内での間接作業ケースID重複時の409エラーを検証する
  - 存在しない間接作業ケースIDでの422エラーを検証する
  - _Requirements: 8.3, 8.4_
