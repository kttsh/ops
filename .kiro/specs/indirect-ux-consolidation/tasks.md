# Implementation Plan

- [x] 1. useIndirectSimulation フックの簡素化
- [x] 1.1 primaryケース自動検出と保存済みデータクエリの実装
  - ケース選択ステート（selectedHeadcountPlanCaseId 等）、setter、dirty追跡を削除する
  - 3エンティティのケースリストクエリからprimaryケースIDを `findPrimaryId` で自動検出する
  - primaryケースIDを元に、人員計画・キャパシティ・間接工数・比率の4つの保存済みデータクエリを並列実行する
  - primaryケース名をケースリストデータから派生ステートとして算出する
  - `canRecalculate` を3つのprimaryケースIDがすべて非nullかで判定する
  - BU変更時はクエリキーの変化によりTanStack Queryが自動リフェッチするため手動リセットロジックを削除する
  - _Requirements: 1.2, 2.1, 2.5, 2.6_

- [x] 1.2 再計算関数の統合実装
  - `recalculate()` 関数を実装し、キャパシティ計算API呼び出し→クライアントサイド間接工数計算→バルク保存→クエリ無効化を順次実行する
  - 成功時にトースト通知「間接工数を再計算しました」を表示する
  - エラー発生時に `toast.error` でエラー通知を表示し処理を中断する
  - `isRecalculating` フラグで実行中状態を管理し、再実行を防止する
  - _Requirements: 5.2, 5.3, 5.5, 5.6_

- [x] 1.3 利用可能年度リストと最終計算日時の算出
  - 保存済みの間接工数データとキャパシティデータの `yearMonth` フィールドから `getFiscalYear()` で年度を抽出し、重複除去・ソート済みの `availableFiscalYears` 配列を返す
  - 間接工数データのうち `source === "calculated"` のレコードのみを対象に `updatedAt` の最大値を `lastCalculatedAt` として算出する
  - データが存在しない場合は `availableFiscalYears` を空配列、`lastCalculatedAt` を `null` として返す
  - _Requirements: 4.1, 8.1, 8.2_

- [x] 2. UI コンポーネントの実装・改修
- [x] 2.1 (P) 計算条件パネルコンポーネントの新規作成
  - 人員計画・稼働時間・間接作業の3つの条件項目を横並びで表示する
  - 各項目にprimaryケース名をテキスト表示し、横にマスタ画面への遷移リンク（→アイコン）を配置する
  - ケース名が未設定の場合は「未設定」を警告色（amber）で表示し、遷移リンクはマスタ画面に向ける
  - 再計算ボタンを配置し、全ケース設定済みで有効、未設定ありで無効、実行中でローディング表示する
  - マスタ画面パスは人員計画→`/master/headcount-plans`、稼働時間→`/master/capacity-scenarios`、間接作業→`/master/indirect-work-cases`
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 2.6_

- [x] 2.2 (P) 統合テーブルコンポーネントの改修
  - 既存の計算結果テーブルのpropsインターフェースを保存済みデータ直接受け取り型に変更する
  - 年度セレクトボックスを横並びのチップ（ボタン）に置き換え、データが存在する年度のみ表示し、アクティブ年度をハイライトする
  - テーブル行構成を変更し、人員数（年合計「-」）・キャパシティ・間接内訳（折りたたみ可能）・間接計・直接（キャパシティ−間接計）の順で表示する
  - テーブル下部に最終計算日時を「ⓘ 最終計算: YYYY-MM-DD HH:mm」形式で表示し、未計算時は非表示にする
  - 保存済みデータが空の場合は「再計算を実行してデータを生成してください」メッセージを表示する空状態を実装する
  - 旧テーブル上部の条件サマリー表示を削除する
  - _Requirements: 3.1, 3.2, 3.3, 3.4, 3.5, 3.6, 3.7, 4.1, 4.2, 4.3, 4.4, 8.1, 8.2_
