# Implementation Plan

- [x] 1. WorkloadChart の高さ動的制御
- [x] 1.1 チャートコンテナの高さをpropsで切り替え可能にする
  - `fullHeight` オプショナルpropsを追加し、trueの場合はチャートコンテナを親要素いっぱいに拡張、falseまたは未指定の場合は従来の固定高さを維持する
  - ResponsiveContainer が親の高さ変更に追従してリサイズされることを確認する
  - ChartFullscreenDialog ボタンの絶対配置が両モードで正常に表示されることを確認する
  - _Requirements: 2.1, 2.2_

- [x] 2. WorkloadPage のviewモード別レイアウト制御
- [x] 2.1 グリッドコンテナのスクロール・高さ制御をviewモードに応じて切り替える
  - `both` モード時: コンテナから高さ制約を外し、コンテンツの自然な高さでスクロール可能にする
  - `chart` / `table` 単独モード時: コンテナを親いっぱいの高さにしてスクロールなしで画面内に収める
  - `chart` 単独モード時: チャートカードにflex拡張クラスを付与し、カード内いっぱいにチャートを表示する。WorkloadChart に `fullHeight` を渡す
  - _Requirements: 1.1, 1.2, 2.1, 2.2, 2.3, 3.1, 3.2_

- [x] 2.2 チャートと凡例パネルの高さを揃える
  - チャートカード内のflexコンテナでチャートと凡例の高さが一致するよう調整する
  - 現状の不整合原因を特定し（padding不一致、h-full の効き方、flex stretch の挙動）、最小限の変更で修正する
  - chart単独モード・bothモードの両方で凡例パネルの高さがチャートと揃っていることを確認する
  - _Requirements: 4.1, 4.2_

- [x] 3. 動作確認と型チェック
- [x] 3.1 全viewモードの目視確認と型安全性の検証
  - `both` モード: チャート+テーブルが画面を超える場合に縦スクロール可能であること
  - `chart` モード: チャートがカード内いっぱいに余白なく表示され、スクロールが発生しないこと
  - `table` モード: テーブルが画面内に収まり、スクロールが発生しないこと
  - 凡例のホバーハイライト・ピン操作が正常に動作すること
  - ビュー切替時にレイアウトが即座に反映されること
  - サイドパネルの開閉でレイアウトが崩れないこと
  - TypeScript のビルドチェック（`npx tsc -b`）でエラーがないこと
  - _Requirements: 1.1, 1.2, 2.1, 2.2, 2.3, 3.1, 3.2, 4.1, 4.2, 5.1, 5.2, 5.3, 5.4_
