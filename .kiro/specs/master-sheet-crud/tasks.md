# Implementation Plan

- [x] 1. Sheet モード管理の共通型と Hook を作成
  - 閲覧・編集・作成・閉じるの4状態を discriminated union で定義し、各モードに必要なデータを型レベルで保証する
  - open/close、閲覧→編集、編集→閲覧（更新後エンティティ受け取り対応）、作成モード開始の遷移メソッドを提供する Hook を実装
  - Sheet の開閉判定（isOpen）をモード状態から導出する

- [x] 2. ビジネスユニットの Sheet 化（先行実装）
- [x] 2.1 BU の Detail Sheet コンポーネントを作成
  - 閲覧モード: エンティティの全フィールド（コード、名称、表示順、更新日時）を読み取り専用で表示し、編集・削除ボタンを配置する
  - 編集モード: 既存フォームコンポーネントを edit モードで再利用し、保存成功時に更新後エンティティで閲覧モードへ遷移する。キャンセル時は変更を破棄して閲覧モードに戻る
  - 作成モード: 既存フォームコンポーネントを create モードで空フォーム表示し、保存成功時に Sheet を閉じる
  - バリデーションエラー発生時はフォーム上にエラーを表示し、現在のモードを維持する
  - 削除操作: 確認ダイアログ表示後にソフトデリートを実行し、成功時に Sheet を閉じる
  - 復元操作: 削除済みレコード表示時に復元ボタンを提供し、成功時に一覧を更新する
  - API エラーはステータスコード別（409 競合、404 不存在、422 バリデーション）にトースト通知する
  - _Requirements: 1.2, 2.1, 2.2, 2.3, 2.4, 2.5, 3.2, 3.3, 3.4, 4.1, 4.2, 4.3, 4.4, 4.5, 5.4_
