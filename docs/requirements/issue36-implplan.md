## 5. cc-sdd Spec 単位

cc-sdd (Claude Code Spec-Driven Development) の Spec は、設計判断やアーキテクチャ決定が必要な改修に適用する。単純なスタイル修正には不要。

### Spec が必要な改修

| Spec ID | 対応Issue | 説明 | 理由 |
|---------|----------|------|------|
| `workload-layout` | #35 | Workloadチャート・テーブルのレイアウト・スクロール制御 | view モードに応じた動的レイアウト制御の設計が必要 |
| `workload-control-panel` | #37 | コントロールパネルの色選択UI改善 | 新しいインタラクションパターン（Popover色選択）の設計が必要 |
| `master-sheet-crud` | #38 | マスター管理画面のSheet化 | 3画面の大規模なUI構造変更。Sheet内の閲覧/編集モード切替、状態管理の設計が必要 |
| `bulk-import-export` | #42 | 案件一括入出力 | 新機能。API設計、データフォーマット、バリデーション戦略の設計が必要 |

### Spec 不要な改修（直接実装可能）

| 対応Issue | 説明 | 理由 |
|----------|------|------|
