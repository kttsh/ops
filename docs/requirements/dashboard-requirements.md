## 1. システム概要

### 1.1 目的

本システムは、ビジネスユニット（部門）ごとのプロジェクト工数・間接作業工数・キャパシティ（生産能力）を時系列で可視化し、リソース配分の意思決定を支援するための操業山積管理システムである。

### 1.2 システム全体像

```
┌──────────────────────────────────────────────────────────────────┐
│                        操業山積管理システム                        │
├──────────────┬──────────────┬──────────────┬─────────────────────┤
│              │              │              │                     │
│  操業山積    │  プロジェクト │  ケース      │   間接作業          │
│  ダッシュ    │  管理        │  スタディ    │   管理              │
│  ボード      │              │              │                     │
│              │              │              │                     │
│ ・チャート   │ ・案件一覧   │ ・案件選択   │ ・人員計画ケース    │
│ ・コントロール│ ・案件登録   │ ・ケース管理 │ ・作業種類管理      │
│  パネル      │ ・案件編集   │ ・工数計算   │ ・比率設定          │
│ ・データ     │ ・案件削除   │  (標準/手動) │ ・間接工数計算      │
│  テーブル    │ ・案件検索   │ ・月別工数   │ ・計算結果保存      │
│              │              │  編集        │                     │
└──────────────┴──────────────┴──────────────┴─────────────────────┘
```

### 1.3 ユーザーロール

| ロール | 主な利用機能 | 関心事 |
|--------|-------------|--------|
| **部門マネージャー** | 操業山積ダッシュボード、間接作業管理 | 部門全体のキャパシティ把握、人員計画策定 |
| **プロジェクトマネージャー** | ケーススタディ、プロジェクト管理 | 案件の工数見積、シナリオ比較 |
| **経営層** | 操業山積ダッシュボード | 部門間リソース配分、間接業務比率の把握 |
| **計画担当者** | 全機能 | データ入力、計算実行、結果確認 |

## 2. ビジネス要件

### 2.1 解決すべき課題

| ID | 課題 | 詳細 |
|----|------|------|
| B-001 | 工数の不可視性 | プロジェクト工数と間接業務工数が一元的に把握できず、実際のキャパシティとの乖離が発生している |
| B-002 | 計画精度の低下 | 間接業務を考慮しないリソース計画により、プロジェクトへの過剰アサインが発生している |
| B-003 | 種別分析の困難さ | 「間接業務」として一括管理されており、どの種別（教育、見積、会議等）に時間を費やしているか不明 |
| B-004 | シナリオ比較の欠如 | 案件の工数見積を複数パターンで比較・検討する手段がない |
| B-005 | 案件タイプごとの負荷把握 | 案件タイプ（EPC、サービス、Study等）ごとの負荷バランスが見えない |

### 2.2 期待される効果

- 間接業務を含めた総工数の可視化によるリソース配分の最適化
- 種別ごとの間接業務分析による業務改善の推進
- ケーススタディによる精度の高い工数見積
- キャパシティと負荷の比較による適正な人員配置
- 複数ビジネスユニット横断でのリソース状況の把握

### 2.3 業務フロー概要

```
1. 基盤データ準備
   ├── プロジェクト情報の登録・更新
   ├── 人員計画ケースの策定
   └── 間接作業比率の設定

2. 工数計算
   ├── ケーススタディによるプロジェクト工数見積
   └── 間接作業工数の自動計算

3. 可視化・分析
   ├── 操業山積チャートによる負荷・キャパシティの可視化
   ├── データテーブルによる詳細値の確認
   └── コントロールパネルによるフィルタリング・条件変更

4. 意思決定
   ├── リソース配分の調整
   ├── 人員計画の見直し
   └── 案件受注判断への活用
```

### 3.2 F-WC-001: 積み上げエリアチャート表示

#### 概要
ビジネスユニット別の工数（案件負荷+間接作業）を時系列で積み上げエリアチャートとして表示する。

#### 表示仕様

| 項目 | 仕様 |
|------|------|
| チャートタイプ | ComposedChart（積み上げエリア+折れ線の複合チャート） |
| X軸 | 年月（YYYYMM形式、YYYY/MM表示） |
| Y軸 | 工数（人時）|
| スタック順序 | 下から順に: 間接作業（ケースごと）→ 案件工数（案件タイプ順） |
| アニメーション | 無効（データ更新時のちらつき防止、パフォーマンス優先） |

#### データシリーズ

**案件工数（エリア）**
- 案件タイプ（ProjectType）単位で集約・色分け
- 案件タイプごとにdisplayOrderで昇順ソート
- 同一stackIdで積み上げ表示
- 半透明表示（fillOpacity: 0.8）

**間接作業（エリア）**
- 間接作業ケース（IndirectWorkCase）単位で表示
- ケースに紐づく作業種類比率（IndirectWorkTypeRatio）から種別内訳を導出可能
- グレー系のカラーパレット
- 半透明表示（fillOpacity: 0.7）

**キャパシティ（ライン）**
- 複数シナリオ対応（capacityScenarioId単位）
- 破線スタイル
- ドットマーカー表示あり
- 青系グラデーションのカラーパレット

#### カラーパレット

| カテゴリ | 配色方針 |
|----------|----------|
| 案件（案件タイプ） | 案件タイプごとに固有色（青系、緑系、黄系など） |
| 間接作業 | グレー系統で統一（種別を明度差で区別） |
| キャパシティ | 青・紫・緑系統のグラデーション |

#### ビジネスルール
- 案件タイプ単位での集約描画（個別案件を描画するのではなく、案件タイプで集約する）
- 間接作業はケース単位で表示
- 表示/非表示はコントロールパネルの設定に連動

### 3.4 F-WC-003: 固定凡例パネル

#### 概要
チャート右側に常時固定表示される凡例パネル。選択月の内訳データを表示する。

#### レイアウト
- チャート右側に固定配置（幅: 288px）
- チャートと同一高さで連動
- スクロール可能なコンテンツ領域

#### 表示項目

| セクション | 内容 |
|------------|------|
| ヘッダー | 選択中の年月（YYYY/MM） |
| 案件タイプ | 色ラベル+名称+工数値 |
| 間接作業 | 色ラベル+ケース名+工数値 |
| キャパシティ | 色ラベル+シナリオ名+値 |
| サマリー | 合計工数、稼働率（対Capacity比） |

#### 表示モード

| モード | トリガー | 動作 |
|--------|----------|------|
| 初期表示 | ページロード | 最新月のデータを表示 |
| クリック固定 | チャート上をクリック | クリックした月のデータを固定表示 |
| ホバー表示 | チャート上をホバー | ホバー中の月を一時表示（固定中は無効） |

#### 案件ドリルダウン
- 案件タイプクリックで所属案件リスト展開
- 案件名+個別工数を表示
- 折りたたみ可能なアコーディオン形式
- 固定表示中のみ有効（ホバー中は無効）

### 3.5 F-WC-004: チャートクリック詳細表示

#### 概要
チャート上をクリックすると、その月のデータを凡例パネルに固定表示する。

#### インタラクション仕様

| 操作 | 動作 |
|------|------|
| エリア/バークリック | 凡例パネルにその月のデータを固定表示（ピンアイコン表示） |
| 同じ月を再クリック | 固定解除、初期表示に戻る |
| ピンアイコンクリック | 固定解除、初期表示に戻る |
| 凡例パネル外クリック | 固定状態を維持 |

#### ホバー動作

| 状態 | ホバー動作 |
|------|------------|
| 未固定時 | 凡例パネルに一時的にデータ表示 |
| 固定時 | ホバー無効（固定データを維持） |
| ホバー解除 | 未固定時は初期表示に戻る |

#### カーソル表示
- 縦線カーソル（破線）で現在ホバー中の月を示す
- ツールチップボックスは非表示（凡例パネルに委譲）

### 3.10 F-WC-009: 表示設定

#### 概要
案件タイプ・キャパシティの表示をカスタマイズする。表示順と配色は報告資料としての意味を持つ重要な要素であり、報告の目的・局面に応じた柔軟なカスタマイズを可能とする。

#### 配置
- サイドパネル > 設定タブ > 表示設定セクション

#### 案件タイプ設定

| 設定項目 | 説明 |
|----------|------|
| 並び順 | チャート上の案件積み上げ順序のカスタマイズ（ドラッグ&ドロップまたは上下ボタン） |
| 色設定 | 各案件タイプの表示色のカスタマイズ（同一色の意図的な割り当てを許容） |

#### キャパシティ設定

| 設定項目 | 説明 |
|----------|------|
| 表示/非表示 | シナリオごとの表示トグル |
| 色設定 | 各シナリオの線色カスタマイズ |

#### カラーブロック（同色グルーピング）

報告目的で、異なる案件タイプに対してあえて同一の色を割り当て、積み上げ順序上で隣接させることで、視覚的な「色の塊（カラーブロック）」を形成するユースケースに対応する。

| 仕様項目 | 説明 |
|----------|------|
| 同色割り当て | 異なる案件タイプに同じ色コードを設定可能（バリデーションで弾かない） |
| 隣接配置 | 並び順を調整し、同色の項目を上下に隣接させることでブロックを形成 |
| 用途 | カテゴリの視覚化（例: 「大型案件群」「メンテナンス系」など報告上の分類を色で表現） |
| 凡例表示 | 同色のエントリは凡例上でも個別に表示される（色は同一だが名称で区別可能） |

#### ビジネスルール
- 表示順・配色は報告上の意味を持つため、ユーザーが完全に制御できること
- 同一色を複数の項目に割り当てる操作を制限しないこと（意図的な運用であるため）
- 変更はリアルタイムにチャートへ反映すること
- 表示設定の永続化はF-WC-015（表示プロファイル管理）と連携すること

### 3.12 F-WC-011: データテーブル表示

#### 概要
案件別・月別の工数データを表形式で表示する。

#### テーブル構造

**行（縦軸）**

| 行タイプ | 説明 | 表示スタイル |
|----------|------|-------------|
| キャパシティ | キャパシティシナリオ別の値 | 背景色: 青系、太字 |
| 間接作業 | 間接作業ケース別の値 | 背景色: 灰色系 |
| 案件 | 案件別の工数 | 背景色: 白 |

**列（横軸）**

| 列タイプ | 説明 | 幅 |
|----------|------|-----|
| 案件名 | 固定列、左ピン固定 | 280px |
| BU | 固定列 | 80px |
| 案件タイプ | 固定列 | 100px |
| 合計工数 | 固定列 | 100px |
| 月別列 | 1月〜12月（選択年の12ヶ月固定） | 各80px |

#### 数値フォーマット
- 0の場合は「-」表示
- 0以外はカンマ区切りの日本語ロケール表示
- キャパシティ超過時は背景色を赤系で警告表示

#### 仮想化
- 大量データ（最大1,000行）に対応するための仮想スクロール
- 行の高さ: 48px固定

### 6.8 F-IW-006: 月次間接工数管理

#### 概要
間接作業ケースに紐づく月別の間接工数を管理する。

#### 入力項目

| 項目 | フィールド名 | 型 | 必須 | バリデーション |
|------|-------------|-----|:----:|--------------|
| ビジネスユニット | businessUnitCode | 文字列 | ○ | 1〜20文字 |
| 年月 | yearMonth | 文字列 | ○ | YYYYMM形式 |
| 工数 | manhour | 数値 | ○ | 0以上（DECIMAL(10,2)） |
| データソース | source | 文字列 | ○ | "calculated" または "manual" |

#### 計算式（source = "calculated" の場合）

```
月別間接工数 = 人員数 × 月間標準時間 × 比率合計

例：
- 人員数: 10人
- 月間標準時間: 160時間
- 教育比率: 0.05 + 見積比率: 0.03 = 0.08
→ 間接工数 = 10 × 160 × 0.08 = 128時間
```

#### パラメータ

| パラメータ | デフォルト値 | 説明 |
|------------|-------------|------|
| 月間標準時間 | 160時間 | 1人月あたりの標準労働時間 |
| 年間標準時間 | 1920時間 | 1人年あたりの標準労働時間 |

#### 年度の判定
- 4月〜翌3月を1年度とする
- 例：2026年4月〜2027年3月 → 2026年度

#### 一括更新
- `PUT /indirect-work-cases/:id/monthly-indirect-work-loads/bulk` による一括upsert

#### 注意事項
- monthly_indirect_work_load テーブルはケース×BU×年月粒度で合計工数を格納する
- 作業種類別の内訳は、indirect_work_type_ratios の比率から表示時に導出する

## 10. 用語定義

| 用語 | 定義 |
|------|------|
| **ビジネスユニット（BU）** | 組織単位（部門、チームなど）。全データはBU単位で管理される |
| **案件** | プロジェクト案件。案件タイプ・BUに紐づき、月別工数データを持つ |
| **案件コード** | 案件を一意に識別するコード（projectCode、最大120文字） |
| **案件タイプ（ProjectType）** | 案件の分類（EPC、サービス、Study等）。project_types テーブルで管理 |
| **案件ケース（ProjectCase）** | 1つの案件に対する複数の工数計画パターン。calculationType で STANDARD/MANUAL を選択 |
| **計算タイプ** | ケースの工数算出方法。STANDARD（標準工数マスタ参照）と MANUAL（直接入力）の2種類 |
| **標準工数マスタ（StandardEffortMaster）** | BU×案件タイプごとに定義された工数分配パターン（進捗率×重み付け） |
| **案件負荷（ProjectLoad）** | 案件ケースの月別工数データ。project_load テーブルで管理 |
| **キャパシティ** | ビジネスユニットの生産能力上限（人時/月） |
| **キャパシティシナリオ（CapacityScenario）** | キャパシティの計画パターン（定時/残業込み等の複数シナリオ） |
| **作業種類（WorkType）** | 間接作業のカテゴリ（教育、見積、会議など）。work_types テーブルで管理。全BU共通マスタ |
| **間接作業ケース（IndirectWorkCase）** | BUごとの間接作業計画シナリオ |
| **間接作業比率（IndirectWorkTypeRatio）** | ケース×作業種類×年度ごとの間接作業割合（0.0000〜1.0000） |
| **月次間接工数（MonthlyIndirectWorkLoad）** | ケース×BU×年月ごとの間接工数合計値 |
| **人員計画ケース（HeadcountPlanCase）** | 人員計画のシナリオ。オプションでBUに紐づく |
| **月次人員計画（MonthlyHeadcountPlan）** | ケース×BU×年月ごとの予定人員数 |
| **月間標準時間** | 1人月あたりの標準労働時間（デフォルト160時間） |
| **操業山積** | 時系列での工数（案件負荷+間接作業）とキャパシティの可視化 |
| **稼働率** | 工数÷キャパシティの比率。100%超はリソース不足を意味する |
| **積み上げエリアチャート** | 複数のデータシリーズを積み重ねたエリアチャート |
| **仮想化テーブル** | 大量データ表示のため、画面表示範囲の行のみDOMに描画する手法 |
| **チャートビュー（ChartView）** | 保存されたチャート表示設定（期間、含有案件、間接作業項目等） |
| **年度** | 4月〜翌年3月を1年度とする（例：2026年度 = 2026年4月〜2027年3月） |
